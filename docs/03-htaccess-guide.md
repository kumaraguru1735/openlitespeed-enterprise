## What is .htaccess and How OLS Handles It

`.htaccess` is a per-directory configuration file originally from the Apache web server. It lets you change server behavior for a specific directory without editing the main server config.

OpenLiteSpeed has a built-in `.htaccess` parser that supports the most commonly used Apache directives. This means you can use `.htaccess` files written for Apache -- including those generated by WordPress, Laravel, Drupal, and other frameworks -- and they will work on OLS with little or no modification.

OLS parses `.htaccess` files at **request time**, not at server startup. This means changes take effect immediately without a server restart.

---

## Auto-Detection (No Restart Needed)

OpenLiteSpeed polls `.htaccess` files for changes approximately every **2 seconds**. When you create, edit, or delete a `.htaccess` file:

- OLS detects the change automatically.
- New rules take effect within a few seconds.
- No server restart or reload is required.

This is one of the biggest advantages of OLS over Nginx, which has no `.htaccess` support at all.

---

## Enabling .htaccess Support

`.htaccess` support must be enabled in the virtual host configuration. In your `vhconf.conf`, make sure the root context includes a `RewriteFile` directive:

```
context / {
    allowBrowse         1
    location            $DOC_ROOT/
    rewrite {
        RewriteFile     .htaccess
    }
}
```

The `RewriteFile .htaccess` line tells OLS to look for and parse `.htaccess` files. Without this, `.htaccess` files are ignored.

You can also enable rewrite logging for debugging:

```
rewrite {
    enable              1
    logLevel            9
}
```

Log levels range from 0 (off) to 9 (maximum detail). Rewrite log entries appear in the virtual host's error log.

## Supported Directives -- Complete Reference

### Rewrite Rules

The most commonly used `.htaccess` feature. OLS supports the full `mod_rewrite` syntax.

**Basic syntax:**

```apache
RewriteEngine On
RewriteBase /
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^(.*)$ /index.php [L]
```

**WordPress example:**

```apache
# BEGIN WordPress
RewriteEngine On
RewriteBase /
RewriteRule ^index\.php$ - [L]
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule . /index.php [L]
# END WordPress
```

**Laravel example:**

```apache
RewriteEngine On

# Redirect Trailing Slashes
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^(.*)/$ /$1 [L,R=301]

# Send Requests To Front Controller
RewriteCond %{REQUEST_FILENAME} !-d
RewriteCond %{REQUEST_FILENAME} !-f
RewriteRule ^ index.php [L]
```

**Drupal example:**

```apache
RewriteEngine On
RewriteBase /
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteCond %{REQUEST_URI} !=/favicon.ico
RewriteRule ^ index.php [L]
```

**Common RewriteCond variables:**

| Variable | Description |
|----------|-------------|
| `%{REQUEST_FILENAME}` | Full local filesystem path to the requested file |
| `%{REQUEST_URI}` | The request URI path |
| `%{QUERY_STRING}` | The query string |
| `%{HTTP_HOST}` | The hostname from the request |
| `%{HTTPS}` | "on" if SSL, empty otherwise |
| `%{REMOTE_ADDR}` | Client IP address |
| `%{HTTP_USER_AGENT}` | Client user agent string |
| `%{HTTP_REFERER}` | Referring page URL |
| `%{SERVER_NAME}` | Server name |
| `%{SERVER_PORT}` | Server port |
| `%{REQUEST_METHOD}` | GET, POST, etc. |

**Common RewriteRule flags:**

| Flag | Meaning |
|------|---------|
| `[L]` | Last rule -- stop processing |
| `[R=301]` | Redirect with 301 (permanent) |
| `[R=302]` | Redirect with 302 (temporary) |
| `[QSA]` | Append original query string |
| `[NC]` | Case-insensitive match |
| `[F]` | Forbidden -- return 403 |
| `[G]` | Gone -- return 410 |
| `[P]` | Proxy (pass to backend) |
| `[NE]` | No escape (don't encode special chars) |

### Redirects

**Redirect (simple):**

```apache
# Redirect a single page (301 permanent)
Redirect 301 /old-page.html https://example.com/new-page.html

# Redirect a single page (302 temporary)
Redirect 302 /promo https://example.com/sale
```

**RedirectMatch (regex):**

```apache
# Redirect all .html pages to .php
RedirectMatch 301 ^/(.*)\.html$ https://example.com/$1.php

# Redirect an entire directory
RedirectMatch 301 ^/blog/(.*)$ https://newblog.example.com/$1
```

### FallbackResource

A simpler alternative to rewrite rules for front-controller patterns:

```apache
FallbackResource /index.php
```

This sends all requests for non-existent files to `/index.php`. It is equivalent to:

```apache
RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^ /index.php [L]
```

To disable a previously set FallbackResource:

```apache
FallbackResource disabled
```

### Authentication

Protect a directory with username/password authentication.

**Step 1 -- Create a password file:**

```bash
# Install htpasswd utility if needed (part of apache2-utils)
sudo apt install apache2-utils

# Create a new password file with user "admin"
htpasswd -c /usr/local/lsws/conf/vhosts/example.com/htpasswd admin
```

**Step 2 -- Add to .htaccess:**

```apache
AuthType Basic
AuthName "Restricted Area"
AuthUserFile /usr/local/lsws/conf/vhosts/example.com/htpasswd
Require valid-user
```

**Require specific users:**

```apache
AuthType Basic
AuthName "Admin Only"
AuthUserFile /usr/local/lsws/conf/vhosts/example.com/htpasswd
Require user admin editor
```

### Access Control

**Modern syntax (Apache 2.4 style):**

```apache
# Allow only specific IPs
Require ip 192.168.1.0/24 10.0.0.1

# Deny all access
Require all denied

# Allow all access
Require all granted
```

**Legacy syntax (Apache 2.2 style, also supported):**

```apache
Order deny,allow
Deny from all
Allow from 192.168.1.0/24
Allow from 10.0.0.1
```

**Block specific IPs:**

```apache
Order allow,deny
Allow from all
Deny from 123.45.67.89
Deny from 98.76.54.0/24
```

### Environment Variables

**SetEnv -- set an environment variable:**

```apache
SetEnv APP_ENV production
SetEnv DB_HOST localhost
```

**UnsetEnv -- remove an environment variable:**

```apache
UnsetEnv REDIRECT_STATUS
```

**SetEnvIf -- set a variable conditionally:**

```apache
# Set "local" var if request comes from local network
SetEnvIf Remote_Addr "^192\.168\." local

# Set based on user agent
SetEnvIf User-Agent "Googlebot" search_bot

# Set based on request URI
SetEnvIf Request_URI "\.gif$" image_request

# Case-insensitive variant
SetEnvIfNoCase User-Agent "msie" is_ie
```

### Caching and Expiration

**ExpiresActive and ExpiresByType:**

```apache
ExpiresActive On

# Default expiration
ExpiresDefault "access plus 1 month"

# By MIME type
ExpiresByType text/html "access plus 1 hour"
ExpiresByType text/css "access plus 1 week"
ExpiresByType application/javascript "access plus 1 week"
ExpiresByType image/jpeg "access plus 1 year"
ExpiresByType image/png "access plus 1 year"
ExpiresByType image/gif "access plus 1 year"
ExpiresByType image/svg+xml "access plus 1 year"
ExpiresByType font/woff2 "access plus 1 year"
ExpiresByType application/pdf "access plus 1 month"
```

### Compression

```apache
# Enable DEFLATE compression
SetOutputFilter DEFLATE

# Or target specific content types
AddOutputFilterByType DEFLATE text/html text/plain text/css
AddOutputFilterByType DEFLATE application/javascript application/json
AddOutputFilterByType DEFLATE application/xml text/xml
AddOutputFilterByType DEFLATE image/svg+xml
```

Note: OLS has built-in Gzip and Brotli compression configured at the server level (`enableGzipCompress`, `enableBrCompress`), which is more efficient. Use `.htaccess` compression rules only when you need per-directory control.

### Headers

**Set response headers:**

```apache
# Security headers
Header set X-Content-Type-Options "nosniff"
Header set X-Frame-Options "SAMEORIGIN"
Header set X-XSS-Protection "1; mode=block"
Header set Referrer-Policy "strict-origin-when-cross-origin"
Header set Strict-Transport-Security "max-age=31536000; includeSubDomains"
Header set Content-Security-Policy "default-src 'self'"

# Cache control
Header set Cache-Control "public, max-age=3600"

# CORS
Header set Access-Control-Allow-Origin "*"
Header set Access-Control-Allow-Methods "GET, POST, OPTIONS"
Header set Access-Control-Allow-Headers "Content-Type"

# Remove headers
Header unset X-Powered-By
Header unset Server
```

**Set request headers (for proxying):**

```apache
RequestHeader set X-Forwarded-Proto "https"
RequestHeader set X-Real-IP %{REMOTE_ADDR}s
```

### Options

```apache
# Disable directory listings
Options -Indexes

# Enable following symbolic links
Options +FollowSymLinks

# Combine multiple options
Options -Indexes +FollowSymLinks -MultiViews

# Enable ExecCGI for a directory
Options +ExecCGI
```

### PHP Configuration

Change PHP settings per-directory without editing `php.ini`:

```apache
# Set PHP values (equivalent to ini_set for PHP_INI_PERDIR settings)
php_value upload_max_filesize 64M
php_value post_max_size 64M
php_value max_execution_time 300
php_value max_input_time 300
php_value memory_limit 256M

# Set PHP flags (on/off boolean values)
php_flag display_errors Off
php_flag log_errors On
php_flag short_open_tag On

# Disable specific PHP functions
php_value disable_functions "exec,passthru,shell_exec,system"
```

### Conditional Blocks

**IfModule -- apply rules only if a module is loaded:**

```apache
<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteBase /
    RewriteRule ^index\.php$ - [L]
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteRule . /index.php [L]
</IfModule>
```

OLS recognizes common Apache module names and maps them to its own features. Supported modules include: `mod_rewrite`, `mod_headers`, `mod_expires`, `mod_deflate`, `mod_setenvif`, `mod_auth_basic`, `mod_authz_core`, `mod_mime`, `mod_ssl`.

**Files -- apply rules to specific filenames:**

```apache
<Files "wp-config.php">
    Require all denied
</Files>

<Files ".htaccess">
    Require all denied
</Files>
```

**FilesMatch -- apply rules to files matching a regex:**

```apache
# Block access to hidden files
<FilesMatch "^\.">
    Require all denied
</FilesMatch>

# Block access to backup and config files
<FilesMatch "\.(bak|config|sql|fla|psd|ini|log|sh|inc|swp|dist)$">
    Require all denied
</FilesMatch>

# Set caching for static assets
<FilesMatch "\.(ico|jpg|jpeg|png|gif|css|js|woff2)$">
    Header set Cache-Control "public, max-age=31536000"
</FilesMatch>
```

### Error Documents

```apache
# Custom error pages
ErrorDocument 404 /errors/404.html
ErrorDocument 403 /errors/403.html
ErrorDocument 500 /errors/500.html

# Or redirect to a URL
ErrorDocument 404 https://example.com/not-found

# Or show a text message
ErrorDocument 503 "Service temporarily unavailable. Please try again later."
```

### Directory Index

```apache
# Set the default page(s) for a directory
DirectoryIndex index.php index.html index.htm

# For a specific directory, use a different index
DirectoryIndex app.php
```

### MIME Types and Handlers

```apache
# Add a new MIME type
AddType application/x-httpd-php .php
AddType application/json .json
AddType font/woff2 .woff2

# Force a MIME type for all files in this directory
ForceType application/json

# Set the default type
DefaultType text/plain

# Add a handler
AddHandler cgi-script .cgi .pl
```

### SSL

```apache
# Require HTTPS for this directory
SSLRequireSSL
```

### ETag

```apache
# Disable ETags (useful for CDN setups)
FileETag None

# Or set to specific components
FileETag MTime Size
```

## Common .htaccess Recipes

### WordPress with pretty permalinks

```apache
# BEGIN WordPress
<IfModule mod_rewrite.c>
RewriteEngine On
RewriteRule .* - [E=HTTP_AUTHORIZATION:%{HTTP:Authorization}]
RewriteBase /
RewriteRule ^index\.php$ - [L]
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule . /index.php [L]
</IfModule>
# END WordPress
```

### Laravel

```apache
<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteRule ^(.*)/$ /$1 [L,R=301]
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteRule ^ index.php [L]
</IfModule>
```

### Force HTTPS

```apache
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
```

### Force www

```apache
RewriteEngine On
RewriteCond %{HTTP_HOST} !^www\. [NC]
RewriteRule ^(.*)$ https://www.%{HTTP_HOST}/$1 [L,R=301]
```

### Remove www

```apache
RewriteEngine On
RewriteCond %{HTTP_HOST} ^www\.(.+)$ [NC]
RewriteRule ^(.*)$ https://%1/$1 [L,R=301]
```

### Security headers bundle

```apache
Header set X-Content-Type-Options "nosniff"
Header set X-Frame-Options "SAMEORIGIN"
Header set X-XSS-Protection "1; mode=block"
Header set Referrer-Policy "strict-origin-when-cross-origin"
Header set Permissions-Policy "camera=(), microphone=(), geolocation=()"
Header always unset X-Powered-By
```

### Complete static asset caching

```apache
ExpiresActive On
ExpiresByType text/css "access plus 1 year"
ExpiresByType application/javascript "access plus 1 year"
ExpiresByType image/jpeg "access plus 1 year"
ExpiresByType image/png "access plus 1 year"
ExpiresByType image/gif "access plus 1 year"
ExpiresByType image/webp "access plus 1 year"
ExpiresByType image/svg+xml "access plus 1 year"
ExpiresByType font/woff2 "access plus 1 year"
ExpiresByType font/woff "access plus 1 year"
ExpiresByType application/font-woff2 "access plus 1 year"

<FilesMatch "\.(css|js|jpg|jpeg|png|gif|webp|svg|woff2|woff|ttf|eot|ico)$">
    Header set Cache-Control "public, max-age=31536000, immutable"
</FilesMatch>
```

## What is NOT Supported (and Workarounds)

The following Apache `.htaccess` directives are **not supported** by OpenLiteSpeed:

| Directive | Workaround |
|-----------|------------|
| `mod_proxy` directives (`ProxyPass`, `ProxyPassReverse`) | Configure proxying in `vhconf.conf` using an extApp context |
| `mod_alias` (`Alias`, `AliasMatch`, `ScriptAlias`) | Use `context` blocks in `vhconf.conf` |
| `<Directory>` blocks | Not applicable in `.htaccess` (use nested `.htaccess` files) |
| `<Location>` blocks | Use `context` blocks in `vhconf.conf` |
| `<VirtualHost>` blocks | These belong in `httpd_config.conf`, not `.htaccess` |
| `mod_userdir` (`UserDir`) | Configure in `httpd_config.conf` |
| `mod_dav` (WebDAV) | Not supported in OLS |
| `mod_include` (SSI, except ESI) | Use the ESI module instead |
| `RewriteMap` | Use RewriteCond/RewriteRule combinations instead |
| `<Limit>` / `<LimitExcept>` | Use RewriteCond with `%{REQUEST_METHOD}` instead |
| `mod_substitute` (`Substitute`) | Use a response filter module |

## Troubleshooting .htaccess Issues

### Problem: .htaccess rules are ignored

1. **Check that RewriteFile is set.** In `vhconf.conf`, the root context must include:

   ```
   context / {
       rewrite {
           RewriteFile .htaccess
       }
   }
   ```

2. **Check file permissions.** The `.htaccess` file must be readable by the server user:

   ```bash
   ls -la /var/www/example.com/html/.htaccess
   # Should show read permissions for the server user
   chmod 644 /var/www/example.com/html/.htaccess
   ```

3. **Check for syntax errors.** Enable rewrite logging:

   ```
   rewrite {
       enable      1
       logLevel    9
   }
   ```

   Then check the virtual host error log:

   ```bash
   tail -100 /var/www/example.com/logs/error.log
   ```

### Problem: Rewrite rules work in Apache but not OLS

- Make sure `RewriteEngine On` is present.
- Make sure `RewriteBase /` is present if your `.htaccess` is in the document root.
- Check that you are not using unsupported `RewriteMap` directives.

### Problem: 500 Internal Server Error after adding .htaccess

- A syntax error in `.htaccess` can cause a 500 error.
- Check the error log for the exact line that failed.
- Temporarily rename the file to isolate the issue:

  ```bash
  mv .htaccess .htaccess.bak
  ```

- Then add rules back one block at a time until you find the offending rule.

### Problem: PHP values not taking effect

- Only `PHP_INI_PERDIR` and `PHP_INI_ALL` settings can be set via `php_value` and `php_flag`.
- `PHP_INI_SYSTEM` settings (like `disable_functions`) can only be set in `php.ini`.
- Check `phpinfo()` to see which settings are changeable from `.htaccess`.

### Problem: Changes take a few seconds to appear

- This is normal. OLS polls `.htaccess` files every ~2 seconds. Wait a moment and try again.
- If you need instant changes during development, you can restart the server:

  ```bash
  sudo /usr/local/lsws/bin/lswsctrl restart
  ```
